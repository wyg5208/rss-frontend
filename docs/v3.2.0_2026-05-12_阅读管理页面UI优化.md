## 任务目标

优化阅读管理（订阅）页面的UI体验，解决顶部空行占用问题，提升两个配置卡片的视觉层次感和交互一致性。

## 问题诊断与分析

### 问题 1: 阅读管理页面顶部存在空行
**现象**: 进入阅读管理页面后，5个功能TAB顶部有一段空白区域，浪费屏幕空间。
**原因**: header中固定渲染了一个 h-[52px] 的空 div，无论是否有标签操作按钮都始终存在。

### 问题 2: 栏目配置区无边界感
**现象**: 固定栏目和RSS源栏目的标题栏背景色与内容区一致，无法清晰区分。
**原因**: 标题栏未设置独立的背景和阴影样式。

### 问题 3: 折叠图标不一致
**现象**: 固定栏目使用 ArrowRight/ArrowDown 图标，RSS源栏目右侧也有箭头，但固定栏目缺少三角形图标。
**原因**: 两个卡片使用不同的折叠图标实现方式。

## 解决方案与实施

### 1. 阅读管理页面空行修复
**文件**: app/subscribe/page.tsx
**方案**: 将空 div 改为条件渲染，仅在标签TAB且有选中标签时才显示

### 2. 栏目配置面板UI统一优化
**文件**: components/TabConfigPanel.tsx
**方案**:
- 统一使用 SVG 三角形图标，通过 CSS rotate-90 控制展开/收起状态
- 标题栏增加渐变背景和阴影，与内容区形成视觉层次
- 保存提示改为状态驱动的闪现消息
- 移除底部冗余提示文字

## 技术实现细节

### 三角形图标实现
使用 SVG 内置图标，通过 `rotate-90` 类名切换方向：
```tsx
<svg className="w-3 h-3 text-gray-500 transition-transform duration-200">
  <path d="M8 5v14l11-7z" />
</svg>
```

### 标题栏立体感
采用 Tailwind 渐变和阴影样式：
```tsx
className="bg-gradient-to-b from-gray-50 to-white shadow-sm"
```

## 经验教训

### 1. 条件渲染消除空元素
固定渲染的空容器即使内容为空白也会占据空间，使用条件渲染可以完全避免此问题。

### 2. SVG图标优于字体图标
对于简单的三角形、箭头等几何图形，使用原生 SVG 可以精确控制大小、颜色和变换，无需额外依赖。

### 3. 渐变营造层次感
通过简单的 from-gray-50 to-white 渐变配合 shadow-sm 阴影，可以在不改变整体配色方案的前提下有效区分标题栏和内容区。
