### 问题 1: 摘要TAB显示内容不正确

**现象**:

## 技术实现细节

### 核心设计决策

#### 1. 摘要字段选择
- **问题**: 应该显示`summary`还是`ai_summary`？
- **决策**: 使用`ai_summary`（AI生成的中文摘要）
- **原因**: 
  - `summary`是RSS源原始摘要，多为英文
  - `ai_summary`是AI生成的中文摘要，符合用户需求
  - 筛选逻辑必须同步修改为筛选`ai_summary`字段

#### 2. 展开/收起交互设计
- **方案A**: 保留"显示全部"按钮
- **方案B**: 点击摘要文字直接展开 ✅
- **选择B的原因**:
  - 节省空间（移动端宝贵）
  - 操作更直观
  - 减少视觉干扰

#### 3. 预加载策略
- **触发点**: 滚动到70%位置
- **加载量**: 每次加载20条
- **智能补充**: 每看10条自动加载10条
- **目的**: 用户无感知加载，流畅阅读体验

### 关键代码片段

#### IntersectionObserver预加载
```typescript
useEffect(() => {
  const observer = new IntersectionObserver(
    (entries) => {
      if (entries[0].isIntersecting && hasMore && !loading) {
        onLoadMore();
      }
    },
    { rootMargin: "70%" }
  );
  
  if (observerRef.current) {
    observer.observe(observerRef.current);
  }
  
  return () => observer.disconnect();
}, [hasMore, loading, onLoadMore]);
```

#### 智能加载逻辑
```typescript
useEffect(() => {
  const viewedCount = articles.length;
  const shouldLoadMore = viewedCount >= lastLoadedCount.current + 10;
  
  if (shouldLoadMore && hasMore && !loading) {
    onLoadMore();
    lastLoadedCount.current = viewedCount;
  }
}, [articles.length, hasMore, loading, onLoadMore]);
```

## 经验教训

### 1. 经验总结

#### 字段语义要明确
- `summary`和`ai_summary`是两个完全不同的字段
- 筛选逻辑和显示逻辑必须使用相同字段
- 命名要清晰表达含义

#### 移动端交互要精简
- 按钮会占用宝贵的垂直空间
- 点击文本本身是更自然的交互方式
- 减少视觉元素，突出核心内容

#### 文档要跟上产品迭代
- 新功能上线必须配套使用手册
- 帮助用户快速理解产品价值
- 降低用户学习成本

### 2. 注意事项

#### AI摘要字段
- 必须确保后端API返回`ai_summary`字段
- 筛选逻辑要针对`ai_summary`而非`summary`
- 前端降级方案：`ai_summary || summary`

#### IntersectionObserver使用
- 必须在useEffect中清理observer（disconnect）
- 依赖数组要完整，避免闭包陷阱
- rootMargin设置为"70%"表示70%位置触发

#### 调试日志管理
- 开发阶段可以使用console.log
- 提交前必须注释掉所有调试日志
- 使用注释而非删除，方便后续调试
