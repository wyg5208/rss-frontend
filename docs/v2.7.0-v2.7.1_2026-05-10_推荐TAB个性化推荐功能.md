## 问题诊断与分析

### 问题 1: 推荐TAB无个性化

**现象**: 推荐TAB等同于"全部"Tab
**原因**: 原实现直接使用筛选API，未调用推荐API

### 问题 2: React Hook规则违规风险

**现象**: 条件性调用Hook会导致运行时错误
**原因**: 需要两个Hook都顶层调用，通过数据选择切换

## 解决方案与实施

### 1. 推荐Hook (`hooks/useArticles.ts`)

**新增Hook**:
```typescript
export function useRecommendedArticles() {
  return useInfiniteQuery<Article[]>({
    queryKey: ["articles", "recommended"],
    queryFn: async ({ pageParam = 0 }) => {
      const path = buildQuery("/rss/articles/recommended", {
        skip: Number(pageParam),
        limit: PAGE_SIZE,
      });
      return api.get<Article[]>(path);
    },
    staleTime: 30 * 60 * 1000,  // 30分钟，与后端缓存一致
    gcTime: 30 * 60 * 1000,
    retry: 1,
    retryDelay: 2000,
  });
}
```

**设计要点**:
- 与`useArticles`返回格式一致，兼容InfiniteQuery
- 30分钟缓存，与后端推荐缓存TTL一致
- 失败重试1次，避免429雪崩

### 2. 主页Tab切换 (`app/page.tsx`)

**修改逻辑**:
1. 引入`useRecommendedArticles` Hook
2. 两个Hook在顶层调用（遵循React Hook规则）
3. 通过`activeTab === "推荐"`切换数据源

```typescript
// 两个Hook都调用（遵循React Hook规则）
const { data: recommendPages, fetchNextPage: fetchNextPageRecommend, ... } = useRecommendedArticles();
const { data: pages, fetchNextPage, ... } = useArticles(filters);

const articles = useMemo(() => {
  if (activeTab === "推荐" && recommendPages) {
    return recommendPages.pages.flat();
  }
  return pages?.pages.flat() || [];
}, [activeTab, pages, recommendPages]);
```

## 经验教训

### 1. React Hook规则必须遵守

条件性调用Hook会导致"Invalid hook call"错误，两个Hook都必须在顶层调用，通过数据选择切换。

### 2. API返回格式一致性

推荐API与列表API返回相同格式，前端无需特殊处理，降低集成复杂度。

### 3. 缓存时间需与后端一致

前端`staleTime`设为30分钟，与后端推荐缓存TTL一致，避免数据不一致。
