## 一、本次修订内容

### 1.1 后端推荐API修复

**问题**：后端推荐API `/rss/articles/recommended` 返回422错误

**根因**：FastAPI路由定义顺序问题，`/recommended` 被 `/{article_id}` 路由捕获

**修复**：将 `/recommended` 路由移到 `/{article_id}` 之前

### 1.2 前端无需修改

本次修复是后端问题，前端代码无需任何修改。

前端 `useRecommendedArticles` hook 已正确实现：
```typescript
// hooks/useArticles.ts
export function useRecommendedArticles() {
  return useInfiniteQuery<Article[]>({
    queryKey: ["articles", "recommended"],
    queryFn: async ({ pageParam = 0 }) => {
      const path = buildQuery("/rss/articles/recommended", {
        skip: Number(pageParam),
        limit: PAGE_SIZE,
      });
      return api.get<Article[]>(path);
    },
    // ...
  });
}
```

## 四、教训总结

### 4.1 后端路由规范

**FastAPI路由顺序规则**：
```
1. 静态路由优先（/recommended, /search）
2. 带路径参数的路由次之（/{id}）
3. 带查询参数的路由最后（/）
```

**教训**：新增路由时应检查是否与现有路由冲突

### 4.2 API测试必要性

**教训**：新增API必须用curl/Postman实际调用测试，不能只依赖单元测试

```bash
# 测试推荐API
curl "http://localhost:8001/api/v1/rss/articles/recommended?limit=3"

# 期望：返回文章列表
# 错误：返回422
```
